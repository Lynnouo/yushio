# 美术总监参考案例库 · Design Case Library

> 这是《美术总监夕潮》SKILL 的伴随参考文件,不是独立 skill(无 frontmatter)。
> 它存放**真实项目的设计提炼**——给美术总监当"见过的样本"。看一个陌生项目要定方向时,先翻这里有没有同形态的参照。
> **一项目一条**,可持续积累。被 SKILL §4.3(范例)+ §9 #DH(反 AI-slop)引用。
> 来源约束:这些案例是从只读学习素材里提炼的,**记录的是设计判断,不是去改那些项目**。

---

## 怎么用

- **定方向前**:扫一遍有没有同**产品形态**的案例(卡牌对战 / dashboard / 工具 / 叙事…),借它的"可迁移要点",而不是抄它的具体值。
- **反 AI-slop**:每条案例的 `banned-patterns` 段 = 那个项目"刻意不做"的清单,是写自己项目 banned-patterns 的种子(配合 §9 #DH)。
- **不是 style guide**:这里记"为什么这样 + 气质方向",不是逐组件 specs。

## 案例 schema(每条照此填)

```
## 案例 N · <项目形态匿名描述>(<产品形态> · <技术栈>)
- 一句话视觉定位
- 视觉 DNA:色 / 字 / 组件 / 动效
- 核心交互或战斗动效语言
- banned-patterns(它刻意不做的)
- 可迁移要点(抽象成跨项目判断)
- 出处 + 诚实标注(取证范围 / 哪些还在变)
```

## 案例 1 · 某卡牌对战游戏(日系 RPG 风 · 零手动操作 PvE · React + TS + Pixi + Spine + Framer Motion)

### 一句话视觉定位

**克制的深色日式 RPG(Restrained dark JRPG)+ 视频驱动的卡点战斗演出。** 形式追随感受:游戏是"零手动操作"的卡牌对战,玩家唯一动作是出牌,战斗全程自动结算 → 玩家是"观众" → 战斗的爽感全压在演出上;而深暗底色,是为了让鲜艳的卡牌和技能特效"跳出来"。参考坐标(项目自陈的气质目标):深底 + 鲜艳点缀的日系治愈/养成美学。

### 视觉 DNA

**色**(真源 = 项目的设计单源文档 + 主题 palette 文件)
- 整套 UI **99% 深暗墨蓝底 + 暗底白字**(单一 hue,与战斗 token 同源)。
- **4 主 accent 总面积 ≤ 10%**:樱粉(系统/推荐)· 日青(在线/制作)· 暖金琥珀(金币/主按钮/最高稀有)· 朱砂(警示/战斗)。
- **稀有度是独立维度**(低→高:草绿 / 深蓝 / 桃粉 / **暖金,不是红**),不与 accent 串。
- 全程 **oklch**;禁 `#000`/`#fff`/裸 hex,黑白都用 oklch + 微染。

**字**(3 字族各司其职,"绝对不准混用")
- 明朝体(serif)→ 标题 / 卡牌名 / 仪式(日式作品的仪式感)
- 无衬线中文 → 中文正文 / 气泡
- 衬线西文(tabular-nums)→ 西文 / 数字(金币 / Lv / Power)

**组件**
- 玻璃/卡片三档分层,但 glassmorphism 禁止当默认。
- 4 档语义按钮(Primary=暖金 · Danger=朱砂 · Secondary=墨 · Ghost=透明)。
- "金币原子":图标 + tabular-nums + 暖金色,全局走同一个金币展示组件,不各拼各的。
- radius / elevation / shadow 全走 token,不在 inline 堆 4 层阴影自创层级。

**动效**
- 5 档 duration(100/200/350/500/800ms);统一 `cubic-bezier(0.16,1,0.3,1)`(ease-out-quart)。
- **UI 切换禁 spring bounce**——只有卡牌 reveal / 抽卡仪式这种"庆祝瞬间"才允许弹。

### 战斗动效语言(卡牌对战 · 视频驱动卡点)

**核心理念**:零手动选目标 → 观众型战斗 → 重演出补回爽感。

**节拍**(战斗状态 store,5 拍 Beat):部署 600ms → 技能演出 1700ms → 羁绊/被动 800ms → 结算公告 600ms → 结算总 3500ms;命中 hitstop 80ms。后期整体基础 ms 砍 ~35%(回应"节奏偏慢")。

**一次技能 = 多层叠合**(战斗演出组件):
1. **caster 立绘中央 banner 一闪而过**(横版脸部特写,face→portrait→cards 三层回退,800ms 出 / 900ms 退)
2. **8 类标准技能视频**(技能视频清单:all_attack / row_attack / buff / shield / summon / silence / chaos…)全屏后台,`startMs` 跳铺垫,`hitMs≈800ms` **卡高潮**
3. **行/半区 overlay 视频**(`mix-blend-mode: screen` 黑底抠透,落在对应行/半场,1500ms)
4. **光链**(SVG 从 caster 射向 target,带方向箭头)
5. **跳字**(±N 浮起+旋转+粒子,1800ms)
6. **caster/受击双层加亮**(caster 绿光脉冲;受击按性质:buff 绿光晕上抬 / debuff 红震动)
7. **SFX** 在 hitMs 同步打

**颜色语义**(战斗专用 token,与 UI palette 分离):我方/增益 = 磷光绿 / 敌方/攻击控制 = 血红;统一 caster 视角(attack→敌方色,buff/shield→我方色);砍黄色简化为"同方绿/跨方红"二元可读。

**z 层级秩序**:棋盘单位 100 < overlay 视频 140 < 全屏斩击 145 < **光链+跳字 150**(曾被头像遮挡,从 40/45 提到 150)< portal 扫描线 1000。入场 stagger 每卡错开 80–100ms。

### banned-patterns(它刻意不做的)

UI(项目设计单源文档的 Banned Patterns 段):
- **N×3 等大 feature 卡网格**(点名"SaaS 反射")· **modal-first**(优先 inline/渐进)· 渐变文字 · side-stripe 装饰条 · glassmorphism 当默认 · 硬编码颜色 · 过度堆叠的纯文化意象(已退役的旧 theme)。

资产(组件铁律):
- 小图必走方形头像组件 · 道具必走道具图标组件 · **禁 emoji / 禁 Unicode 字符当图标占位符 / 禁把大卡图缩成圆头**。

战斗(沉浸优先的减法):
- 删每回合 AI 点评浮气泡(判定破坏沉浸)· 删局间黑屏 · 删 CSS 横扫换视频(不够好看)。

### 可迁移要点(抽象成跨项目判断)

1. **深色克制底 + 小面积鲜艳 accent** = 让"主角内容"(卡牌/特效/数据)跳出来的通用手法;accent 面积预算化(≤10%)是把"克制"变成可执行的数字。
2. **稀有度/价值维度与功能 accent 解耦** = 任何带"资产价值语言"的产品(卡牌/装备/段位/会员等级)都该独立成一套色,不与"主按钮/警示"混。
3. **形态决定演出权重**:交互越"被动/观众化",越要把爽感堆在演出上;交互越"主动/高频",越要克制演出别挡操作。这个项目是前者极端 → 重演出成立。
4. **视频驱动卡点**的取舍:单镜头视觉力度强(电影感),代价是**体积大 + 卡点对齐脆弱**(演出时长必须 ≥ 视频时长 + 缓冲,否则"没播完就切")。适合低频高光时刻,不适合每秒触发的高频反馈。
5. **二元颜色语义**(我方绿/敌方红)= 自动结算战斗里"瞬间可读敌我"的最低成本方案。
6. **banned-patterns 写成清单 + 机器护栏** = 反 AI-slop 从"靠品味"变成"会报错"(配合 §9 #DF / #DH)。

### 出处 + 诚实标注

- 取证:项目设计单源文档(视觉真相全文)+ 战斗特效实现扫描(演出组件 / 技能视频清单 / 战斗状态 store)+ 演出演进史日志。
- **未逐行读**主题 palette 源码,个别具体 oklch 值以源码为准。
- 战斗演出仍在**高频迭代**,部分处于"用户嫌丑待迭代"状态——**不是定稿**。
- 该案例提炼自一个由非程序员策划 + Claude Code 密集做出的项目(只读学习素材)。

## 案例 2 · 某 RPG 风格项目(对话驱动叙事化战斗 · React + TS · 多 AI 工具混合美术线)

### 一句话视觉定位

**「美术总监巡视全量资产」的可复用模式 = 单文件 html 资产清册站**。该项目视觉风格本身仍演变中(沿用案例 1 提炼的暗色克制 RPG 底色),本案例**重点不是该项目设计 DNA**,而是它意外发明的**资产巡视模式**——已抽象成跨项目 [`asset-inventory-pattern.md`](./asset-inventory-pattern.md)。

### 视觉 DNA(沿用案例 1 简略复用)

简略复用要点:99% 深暗墨蓝底 + 4 主 accent ≤ 10% / 明朝体 serif + 无衬线中文 + 衬线西文三族不混用 / 5 档 duration + 单一 ease-out-quart 曲线。

该项目在案例 1 之上的演化:
- **战斗演出删除**(改为叙事化战斗——AI 小说战报驱动,不走视频驱动卡点)
- **资产规模翻倍以上**(数百怪物 + 多层地图 + 数十区域背景 + 数个商店背景)
- **多 AI 工具混用**(Nano Banana / Gemini 3 Pro Image / rembg 等)→ 视觉一致性挑战上升

### 资产巡视语言(本案例核心创新)

项目美术 session 落地的资产巡视 html 是「美术现状的一页」——单文件 html,4 段 sticky nav,内联 CSS + 极简 JS,Lightbox 缩放。**不是** audit dashboard,**是**美术总监跨 session 视觉巡视的载体。

**为什么会发明**:某次美术批量产出 session 完成数百只怪物全量入库(从案例 1 的几十张暴增 5+ 倍)+ 多层地图 + 数十区域背景 + 商店背景。规模到达「无法用 Finder cover flow 一次性扫完」临界点。项目 owner 需要「打开就能巡视全部 / 不启服务 / 不引框架 / 跨 session 反复来看」的工具。

**它做了什么 7 件事**(已抽象成 7 原则 · 详见 [`asset-inventory-pattern.md`](./asset-inventory-pattern.md) §3):
1. 单文件零依赖
2. 缺图 onError 半透明灰度兜底(不静默隐藏 · 让缺漏可见)
3. 真源行号 inline 在每段 sec-meta(`<真源 CSV>` / 待建段标 "待生成")
4. 业务分类用色 chip(area-wild / area-dungeon / area-boss / tier-elite / tier-boss)
5. 工艺溯源 chip(模型/工具名)记录每张资产用的 AI 模型
6. 三种 grid 形态共存(怪物 130px 圆勋章 + 区域 380×214 宽卡 + 层地图 240×240 方卡)
7. Lightbox + 双版本预览(grid 看透明底 `-tp.webp` 辨识 · Lightbox 看暗底 `-ondark.webp` 真实表现)

### banned-patterns(它刻意不做的)

- 不引 React / Vue / 任何前端框架 → 失去「双击即开」
- 不加上传 / 删除 / 编辑按钮 → 那是 admin panel
- 不加 BGM / slideshow → 那是 portfolio
- 不用 emoji 做分类(👑 / 🔥 / ⚔)→ §3.2 AI slop 指纹 + 跨平台漂移
- 不静默隐藏缺图(`display:none`)→ 失去缺漏警示 · 用半透明灰度兜底
- 不把生成逻辑写进 html → 生成器分离(`scripts/_gen_<asset>_review.{py,cjs,mjs}` 等)

### 可迁移要点(抽象成跨项目判断)

1. **资产规模 > 100 + 多 AI 工具协作**时,单文件 html 资产清册站是低成本高复用的巡视工具
2. **缺图 onError 兜底半透明灰度**(不是 `display:none`)= 资产层 #DK 防御——让缺漏可见而不假齐全
3. **业务分类用色 chip 不是 emoji** = 反 AI slop 在资产清册场景的落地
4. **AI 时代特有的工艺溯源 chip** = #DG 视觉模式漂移的资产层防御工具
5. **三种 grid 形态共存按资产天然比例匹配** = 拒绝「统一卡尺寸」的 Pinterest 强迫症
6. **生成器(`.py` / `.cjs` / `.mjs`)和产物(`.html`)分离** = 让重生有低摩擦入口 · html 始终可独立打开

### 出处 + 诚实标注

- 取证:某 RPG 风格项目美术 session 产出的资产巡视 html(单文件)+ 配套生成器实现(Python 扫 CSV + glob 资产路径)
- 提炼路径:dogfooding → 抽象成 [`asset-inventory-pattern.md`](./asset-inventory-pattern.md) 跨项目 pattern + 出 [`asset-inventory-starter.html`](./asset-inventory-starter.html) 通用模板
- 该项目设计 DNA 本身**仍在演变**——本案例不抄具体视觉值 · 只抄它新发明的「资产巡视模式」
- 该项目美术资产**仍在补充期**(部分类资产如装备 / 道具 / NPC 立绘尚未批量生成),清册站会持续扩段
